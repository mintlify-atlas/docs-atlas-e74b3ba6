---
title: Request Approval
api: POST /api/boards/{board_id}/approvals
---

Create a new approval request for a board action.

## Authentication

Requires admin or agent authentication with write access to the board.

## Path Parameters

<ParamField path="board_id" type="string" required>
  UUID of the board
</ParamField>

## Request Body

<ParamField body="action_type" type="string" required>
  Type of action requiring approval (e.g., `tool_use`, `task_assignment`, `configuration_change`)
</ParamField>

<ParamField body="confidence" type="number" required>
  Confidence score from 0 to 100
</ParamField>

<ParamField body="status" type="string" default="pending">
  Initial status: `pending`, `approved`, or `rejected`
</ParamField>

<ParamField body="task_id" type="string" optional>
  Primary task UUID associated with this approval
</ParamField>

<ParamField body="task_ids" type="array" optional>
  Array of task UUIDs if multiple tasks are involved
</ParamField>

<ParamField body="agent_id" type="string" optional>
  Agent UUID requesting the approval
</ParamField>

<ParamField body="payload" type="object" optional>
  Action-specific data and context
</ParamField>

<ParamField body="lead_reasoning" type="string" required>
  Explanation of why approval is needed. Can also be provided in `payload.reason`
</ParamField>

<ParamField body="rubric_scores" type="object" optional>
  Evaluation scores for different rubric dimensions
</ParamField>

## Response

Returns the created approval object.

<ResponseField name="id" type="string">
  Approval UUID
</ResponseField>

<ResponseField name="board_id" type="string">
  Board UUID
</ResponseField>

<ResponseField name="action_type" type="string">
  Type of action requiring approval
</ResponseField>

<ResponseField name="status" type="string">
  Current status: `pending`, `approved`, or `rejected`
</ResponseField>

<ResponseField name="confidence" type="number">
  Confidence score (0-100)
</ResponseField>

<ResponseField name="task_id" type="string" optional>
  Primary task UUID
</ResponseField>

<ResponseField name="task_ids" type="array">
  Array of associated task UUIDs
</ResponseField>

<ResponseField name="task_titles" type="array">
  Array of task titles
</ResponseField>

<ResponseField name="agent_id" type="string" optional>
  Agent UUID
</ResponseField>

<ResponseField name="payload" type="object" optional>
  Action payload
</ResponseField>

<ResponseField name="rubric_scores" type="object" optional>
  Rubric scores
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp
</ResponseField>

<ResponseField name="resolved_at" type="string" optional>
  ISO 8601 timestamp when resolved
</ResponseField>

```bash
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{

    "confidence": 85.5,
    "status": "pending",

      "command": "kubectl apply -f deployment.yaml"
    },

      "safety": 90,
      "necessity": 85
    }
  }'
```

## Approval Workflow States

### Pending

The approval is awaiting human review. Only one pending approval is allowed per task to prevent conflicts.

### Approved

The action has been approved and the agent can proceed. The board lead is notified via gateway message.

### Rejected

The action has been rejected. The board lead is notified and the agent should not proceed.

## Error Responses

### 409 Conflict

Returned when attempting to create a pending approval for a task that already has a pending approval.

```json
{
  "detail": {
    "message": "Each task can have only one pending approval.",
    "conflicts": [
      {
        "task_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
        "approval_id": "existing-approval-id"
      }
    ]
  }
}
```

### 400 Bad Request

Returned when `lead_reasoning` is missing or invalid.

```json
{
  "detail": "lead reasoning is required"
}
```
